# 工作说明

设计目标：一个通用的聊天室服务器

## 关键设计原则

1. 整套系统基于NodeJS环境，推荐使用 pnpm 进行多包管理
2. 整套系统基于 WebSocket 和 HTTP 接口，其中 WS 为聊天室核心，支持连接后进行操作，HTTP主要提供查询服务和控制台服务

## 需要支持的功能

1. 每个客户端可以连接上服务器，成为一个独立的用户客户端对象，带有唯一的注册id
2. 用户可以选择创建、加入群。
3. 群有自身的权限控制，创建群的用户会成为群主，群主可以设置群管理员（上限为4个）。
4. 群主及管理员可以设置群公告
5. 只有群主可以删除群
6. 用户在加入群后可以在群内发言
7. 用户之间可以私聊，私聊可以视作无法邀请人的两人群

## 持久化

可以先基于内存来管理这些数据，但请设计好抽象部分，方便后续接入缓存和持久化数据库
